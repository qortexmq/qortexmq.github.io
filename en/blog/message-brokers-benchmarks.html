Message Broker Performance Benchmarks

Comparative analysis of QortexMQ, Kafka, RabbitMQ and Redis in various scenarios

Testing Methodology

  • Environment: AWS c5.large (2 vCPU, 4 GB RAM)
  • OS: Ubuntu 20.04 LTS
  • Test scenarios: 1M messages of 1KB each
  • Metrics: Throughput, Latency, Resource usage
  • Versions: QortexMQ 1.3.0, Kafka 3.4, RabbitMQ 3.12, Redis 7.0

Conclusions and Recommendations

‚úÖ Choose QortexMQ if:

  • Developing a new project or startup
  • Team of up to 10 developers
  • Limited infrastructure budget
  • Need fast setup and prototyping
  • Up to 100M messages per day

üöÄ Choose Kafka if:

  • Processing over 1B messages per day
  • Need long-term message storage
  • Have DevOps team for maintenance
  • Enterprise reliability requirements

‚ö° Final Performance Ranking

  1. Apache Kafka - maximum performance, enterprise features
  2. Redis Pub/Sub - best latency, but limited functionality
  3. QortexMQ - optimal balance of performance and simplicity
  4. RabbitMQ - rich functionality, but high latency
